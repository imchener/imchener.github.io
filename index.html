浅推科学上网原理

今天上计算机网络课时，讲到了虚拟专用网络 VPN 和网络地址转换 NAT，其实一直很好奇那些“机场”是怎么为用户提供“ FQ ”服务的。所以打算试着推测一下 FQ 的原理，不知道对不对，但还是先记下来。

下面以出国为例，回国原理应该也一样。

首先，假设公司 A 向用户 a, b, c 提供 FQ 业务，A 此时需要向 ISP 申请可用于海外通信的线路，然后构建虚拟专用网。这里要注意一点，a, b, c 购买了 A 的业务后，A 就把 a, b, c 当作是员工而不是客户了，因为 a, b, c 能使用 A 组建的虚拟专用网，而客户没有这种权限。

D 类地址向以太网多播地址的映射的目的

将 D 类地址向以太网多播地址映射的目的其实与引入多播组的标识符的目标是一样的。

在 IP 网范围内需要一个群标识，在局域网中传播仍需要一个标识。原本只要讲 D类 IP 地址直接放到 28 为 MAC 地址的后半段就行了，可惜 RA 太“抠”了，会将分配的 MAC 地址的前 24 位中的 23 位给确定了（除去 I/G 位），因此 48 位只剩下 24 位可用了。

又不知道什么原因，25 位只能是 0，故能用的只剩下 23 位了。但 D 类地址抛去首部标识还有 28 位，这样在向 23 位的 MAC 地址映射时就不是一一对应了，而是 2^5 : 1。但这也没办法，在路由器中用软件过滤掉不是要接受的数据包就行了。
