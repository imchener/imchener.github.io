前言

  作为面向社会化编程的希望，Git 是每个开发者都要学习的。

作战武器之 Git 常用指令

  • git init

    • 在我们的工作目录（工作区）下，创建 Git 版本库（.git 隐藏目录），并且 Git 会自动为我们在版本库中创建唯一一个 master 分支，下面介绍的 git reset 命令会用到。

  • git status

    • 查看本地 Git 仓库的文件修改状态

  • git add file_name

    • 把文件加入到版本库中，实际上是将文件修改加入版本库下的暂存区中。

  • git commit -m “注释”

    • 把暂存区的**所有内容（所有的文件修改）**提交到当前分支（版本库包含暂存区和分支）。
    • 其中，默认分支是我们执行 git init 时，Git 为我们自动创建的唯一一个 master 分支。
    • 关于 commit 的其他操作：参考文章
    • 这样一来，这个文件就进入了版本管理系统的管理之下，今后的文件修改管理都交由 Git 进行。

  • git log

    • 查看提交日志
    • 英文状态下按 Q 退出查看状态

  • git push

    • 更新 GitHub 上的远程仓库

  • git chectout – file_name

    • 在使用 git add 命令将文件修改从工作区提交到暂存区提交之前，丢弃指定文件的文件修改。
      • 例如在本地对 README.md 进行修改，在 git add 将文件修改提交到暂存区之前（此时 git status 时文件修改提醒会变成红色），执行该命令。此时文件会更新为我第一次提交文件修改时文件的状态。
      • 进一步实验，在本地对 README.md 进行修改，并且 git add 后（此时 git status 时文件修改提醒会变成绿色），执行该命令。本地文件并不会再回退，此时需要 Git 提供的 git reset 命令。

  • git reset HEAD file_name

    • 把暂存区的修改撤销掉（unstage），回退到工作区，HEAD 代表的是最新的版本。
      • 这样在进一步实验中提交的文件修改就会从暂存区中回退到工作区。
      • 注意回退不是修改，Git 只是把存在在工作区中的文件修改给撤销了。此时仍然需要在工作区执行 git checkout – README.md 操作。这样才能更新文件到第一次提交分支时的状态。

  • git chectout 和 git reset HEAD file_name 的使用总结（版本回退的依据待证明）

    • 当 git add 文件修改后，文件版本回退的依据是版本库中的分支（默认是 master）。
      • 在对文件修改 git commit 之前，可以先后执行 git reset HEAD file_name 和 git checkout – file_name 这两个指令，将我们的文件更新成我们最近一次 git commit 将文件修改添加到分支后的文件修改。
    • 在 git add 文件修改前，文件版本回退的依据是版本库中的暂存区（stage），因为此时还有添加文件修改到分支上。
      • 在对文件修改 git add 之前，只需要 git checkout – file_name 就可以了，此时文件会更新成我们最近一次 git add 将文件修改添加到工作区后的文件修改。
    • 在 git commit 文件之后，文件版本回退的依据依旧是版本库汇总的分支（默认是 master），只不过需要执行下面的 git reset --hard commitld 命令。

  • git reset --hard commitId

    • 如果将文件修改 git commit 后，想要撤回，可以使用下面的方法。
    • 首先使用 git log 查看 git 提交历史,并选取自己要返回的 commitId。
    • 接着 git reset --hard commitId，根据 commitId 来进行撤销。
      • 指定版本撤销：git reset --choose_commitld
      • 返回到上个版本：git reset --hard HEAD^
      • 返回到n个版本,n为正整数：git reset --hard HEAD~n
    • 此时会自动更新成指定版本的文件修改，不再需要 git checkout – file_name 操作。

  • git reset 使用总结

    • git reset 命令既可以回退版本（git reset --hard commitId），也可以把暂存区的修改回退到工作区（git reset HEAD file_name）。

作战领域之 Git 的三大重要区域

  • 工作目录（Working Directory）
    • 我们的本地 Git 仓库。
  • 版本库（Repository）
    • 本地的隐藏目录 .git。
    • .git 包含暂存区（stage 或 index）和分支（包含唯一一个 master 分支）。
  • Git 远程仓库
    • 我们在 GitHub，Gitee 等代码托管平台上创建的仓库。

作战方针之 Git 管理的是文件修改

  • 当我们修改了文件，而没有提交文件修改时，git status 会提醒我们该文件修改未添加到暂存区。
  • 当我们提交了文件修改，但没有将文件修改添加到分支时，git status 会提醒我们可以回退版本。

作战方针之 Git 不管理文件的添加

  • 对于版本库中的文件（经过 git add 命令操作后的文件）
    • Git 管理的是：文件的删除（如 deleted: README.md）、文本内容的更新（如 modified: README.md）、文件的版本回退。
    • 文件添加是我们在工作目录下进行的，由我们自己管理。如果在文件目录下新建一个文件，Git 会提醒我们该文件还未添加到版本库中，即还未追踪该文件的文件修改。
    • 这段确实有点废话了，但一定要理解什么是管理文件修改。
